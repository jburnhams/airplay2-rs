# Contributing to airplay2-rs

Thank you for your interest in contributing to airplay2-rs! We welcome contributions from the community.

## Getting Started

1.  Fork the repository.
2.  Clone your fork: `git clone https://github.com/your-username/airplay2-rs.git`
3.  Install dependencies (Rust toolchain).

## Development Flow

1.  Create a new branch for your feature or bugfix.
2.  Write code and tests.
3.  Ensure all tests pass: `cargo test --all-features`
4.  Ensure code is formatted: `cargo fmt`
5.  Ensure clippy is happy: `cargo clippy --all-features --all-targets`
6.  Submit a Pull Request.

## Code Style

-   Follow standard Rust idioms.
-   Use `cargo fmt` to format your code.
-   Document public APIs.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub.
